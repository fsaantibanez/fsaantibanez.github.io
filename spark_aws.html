Descripción¶

Yelp es un directorio de servicios a nivel mundial, que permite a sus usuarios el evaluar los servicios (restaurants, bancos, clnicas, gimnasios, entre otros) para encontrar y sugerir mejores servicios.

Para esta prueba utilizaremos los datos disponibilizados por Yelp para:

  • Identificar usuarios molestosos.
  • Probabilidad de cierre de los negocios.

Los datos se disponibilizaron en la pgina https://www.yelp.com/dataset/challenge.

Para efectos prcticos de la prueba:

  • Los archivos disponibles se encuentran en el bucket del mdulo, con la direccin s3://bigdata-desafio/yelp-data/.

  • Dentro de esta direccin encontrar distintos archivos json con:

    • Registros respecto al negocio (business.json)
    • check-ins del usuario en un negocio (checkin.json)
    • Fotos asociadas al review (photo.json)
    • Reseas de un usuario sobre el servicio (review.json)
    • Sugerencias del usuario sobre el servicio ( tip.json )
    • Informacin del usuario (user.json )

La definicin de la estructura de datos de cada json se encuentra alojada en la siguiente direccin provista por Yelp: https://www.yelp.com/dataset/documentation/main. (Dentro de este link se encontrarn las definiciones de las columnas y el tipo de registro).

Ejercicio 1: Identificando usuarios molestosos¶

Desde Yelp estn interesados en identificar a aquellos usuarios que se pueden considerar como molestosos. Para ello, tienen la siguiente definicin de un usuario molestoso:

  1. Un usuario molestoso es aqul que su promedio de evaluaciones es menor o igual a 2.
  2. Un usuario molestoso es aquel que tiene en promedio menos de 100 reviews.
  3. Un usuario molestoso es aqul que tiene cero fans.

A partir de esta definicin, se le solicita los siguientes puntos:

  • Identifique en una variable dummy todos los usuarios que se puedan clasificar como molestosos acorde al criterio.

  • Recodificaciones en el archivo user.json:

    • friends, que corresponde a un string con todos los user_id de otros usuarios que siguen al usuario. El objetivo es contar la cantidad de amigos existentes.

    • elite, que corresponde a un string con todos los aos en los que el usuario fue considerado como un reviewer de elite. El objetivo es contar la cantidad de aos en los cuales se consider como elite.

    • Asegrese de eliminar los siguientes registros: friends, yelping_since, name, elite, user_id.

Dado un mejor desempeo del AUC en el modelo GBTClassifier, este debera ser el usado para predecir los usuarios molestosos.

Ahora bien, puede que exista multicolinealidad entre features e influya en obtener un AUC muy alto y a su vez, varios de estos tengan una ponderacin ms cargada.

Revisando la visualizacin de datos, es de esperar que varios features de compliment pesaran poco, puesto que cerca del 75% de sus datos oscilan alrededor del cero, con ello, habra que redefinir los atributos y correr nuevamente el modelo.

Otra opcin es para estas variables que tienden a cero, normalizarlas con un logaritmo y ajustando el valor por un escalar 0.01 para que no se indefinan los registros 0.

Adems existe desbalanceo entre las clases del vector objetivo.

Ejercicio 2: Identificando la probabilidad de cierre de un servicio¶

Desde Yelp estn interesados en predecir la probabilidad de cierre de un servicio en base a los reviews y caractersticas de un negocio. As, la primera iteracin del modelo es generar una identificacin de qu factores estn asociados al cierre.

El equipo de desarrollo de Yelp le hace entrega de un archivo llamado recoding_business_schema.py que describe:

  • Atributos a recodificar

                .otherwise(0).alias('insurance'))

# 1 if business is age-restricted, 0 otherwise
business=business\
    .withColumn('all_ages_allowed', 
        when((col('attributes.AgesAllowed') == 'allages')\
            | (col('attributes.AgesAllowed') == "\'allages\'")\
            | (col('attributes.AgesAllowed') == "u\'allages\'"), 1)\
        .otherwise(0).alias('all_ages_allowed'))

# 1 if business allows alcohol consumption, 0 otherwise
business=business\
    .withColumn('alcohol_consumption', 
        when((col('attributes.Alcohol') == 'beer_and_wine')\
             | (col('attributes.Alcohol') == "\'beer_and_wine\'")\
             | (col('attributes.Alcohol') == "u\'beer_and_wine\'")\
